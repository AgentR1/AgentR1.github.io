核心判断：领域 Agent 越专业，越依赖知识获取

当前大模型智能体正在从“回答问题”走向“解决问题”。在科学研究、医学诊疗、材料发现、化学反应预测、 时序预测和科技文献挖掘等专业领域，任务对知识、证据和时效性的依赖更强。 专业 Agent 不能只依赖参数化记忆，因为科学知识持续更新，论文结论存在边界条件， 不同方法也有清晰的适用场景。

因此，未来领域 Agent 的核心能力之一，不是“知道多少知识”，而是能否在任务执行过程中持续获取、 筛选、组织和使用专业知识。Paper Search Agent 在这里承担“知识感知器”和“证据供给层”的角色， 为上层 Agent 的任务规划、方法设计、实验决策、模型选择、结果解释和论文写作提供最新、可追溯的科学依据。

Paper Search Agent 是 Agentic AI 时代的科学知识入口。 它将开放文献世界中的论文、方法、实验和引用关系，转化为上层智能体可调用、可验证、可推理的结构化证据， 从而支撑专业 Agent 的自主研究、领域决策和持续进化。

核心定位：从论文搜索工具到知识获取基座

传统 paper search 主要解决“给定 query，返回相关论文列表”。这种模式更接近信息检索系统， 关注关键词匹配、语义匹配、排序和召回。面向 Agentic AI 的 Paper Search Agent 需要解决更复杂的问题： 给定一个任务状态，Agent 当前缺少什么知识？应该检索哪些证据？如何沿论文、引用、方法、数据集和实验结果展开搜索？ 如何判断证据是否可靠？如何把证据转化为上层 Agent 可用的结构化知识？

Before

服务人类读论文

过去的文献检索工具主要面向人类研究者，目标是帮助人更快找到论文、浏览摘要、追踪引用和整理相关工作。

Now

服务 Agent 获取知识

新的目标是让智能体具备持续获取科学知识的能力，把论文列表升级为可直接参与推理、规划和决策的 evidence object。

三层能力架构

Paper Search as Agentic AI 基建可以拆成三层：底层提供可调用的检索 API， 中层把论文结果转化为结构化证据，上层把知识获取过程嵌入 Agent Runtime。 这样上层 Agent 不需要自己处理复杂检索细节，而是围绕任务状态调用专业知识获取能力。

Paper Search API Layer

提供 search、expand、rerank、filter、deduplicate、citation tracing、forward / backward citation search、 related work discovery、author / venue / dataset search、code / resource search 等基础能力。

Evidence Structuring Layer

把论文标题和摘要进一步抽取为 paper card、claim、method、dataset、experiment、limitation、 citation relation、implementation clue 和适用场景。

Agentic Knowledge Acquisition Runtime

根据上层 Agent 的任务状态判断知识缺口，生成检索计划，执行多轮搜索与扩展，阅读关键论文， 组织证据链并把结果反馈给任务执行流程。

一个典型调用方式

当 NeoResearch 在模型设计阶段缺少架构启发，CastClaw 面对强非平稳时序数据需要寻找处理策略， 或 Science Agent 生成实验方案需要查找已有对照方法时，它们都可以调用 Paper Search Agent： 先定位知识缺口，再形成检索路径，最后返回可被下游推理使用的结构化证据链。

PaperScout：从 Agentic Paper Search 到基础设施

PaperScout 可以被看作 Paper Search as Infrastructure 的第一步。它的关键贡献在于， 把学术论文搜索从静态语义匹配和固定工作流，转化为一个多轮序列决策过程。 Agent 根据已有 paper pool 动态决定何时 Search、何时 Expand、如何调整搜索方向， 而不是机械执行预设流程。

PSPO 将完整交互序列作为优化单元，用 sequence-level 的方式训练 Agentic Search 行为， 缓解 token-level 优化和 agent-environment interaction 之间的粒度错配。长期看， PaperScout 提供的是 agentic search engine 的雏形，而 Paper Search Agent 要继续扩展到 evidence extraction、paper reading、citation reasoning、claim verification、method mining、 domain adaptation、task-aware search 和 long-term paper memory。

  • 1
    Search 围绕研究问题生成检索查询，建立初始 paper pool，并持续感知当前证据缺口。
  • 2
    Expand 沿引用、被引、相关工作、作者、数据集与方法关系扩展候选集合，主动追踪研究谱系。
  • 3
    Decide 判断哪些论文值得深入阅读、何时停止搜索、如何把候选结果整理成可交付证据。

作为 Agent Runtime 的通用 Skill

复杂 Agent 系统通常会同时包含 Coding Skill、Web Search Skill、Data Analysis Skill、 Paper Search Skill、Experiment Skill 和 Memory Skill。Paper Search Agent 在这里负责专业知识更新， Coding Skill 负责实现，Experiment Skill 负责验证，Memory Skill 负责沉淀经验。

这意味着 Paper Search Agent 不是普通搜索引擎的封装，而是专业任务中的 evidence engine： 它理解任务意图、拆解检索目标、形成多条检索路径、判断哪些论文值得深入阅读， 并沿引用网络和方法谱系组织可验证证据。
