项目简介

在 AI4Science 场景中，科技文献处理远不止“搜一下论文”。研究者需要筛选、比对、追踪引用、组织证据并串联研究脉络； Agent 则需要把这些动作转化为可调用、可决策、可复用的能力。

PaperSearch 将这些能力组织成同一套系统：既服务研究者的日常文献发现与调研，也服务 Claude Code 等 Agent 的工具调用与科研任务执行。 它把文献检索从独立工具推进到基础设施、算法和应用系统共同构成的科研能力栈。

Human-centric 与 Agent-centric

PaperSearch 的核心组织方式是同时面向“人”和“Agent”建设科技文献能力生态。 两条路线并不是彼此割裂的产品线，而是共用同一套文献对象、检索接口、引用关系和能力封装。

Human-centric

让研究者真正把科技文献搜索用起来

Lewen-API 提供开发者能力底座，Lewen-Agent 与 学术乐问 则把这些能力组织成研究者可以直接体验的文献检索 Agent。

Agent-centric

让 Agent 真正把科技文献能力调起来

academic-search 将学术搜索封装为 Claude Code 可加载的科研 Skill； Lewen-API MCP 与 学术鲁班 继续向 Agent 生态开放能力管理与分发。

基础设施：让文献检索可服务于系统

Lewen-API 承担科技文献检索基础设施角色，统一论文对象、检索对象与引用关系， 支持语义检索、关键词过滤以及引用/被引用关系建模。

技术栈采用 SQLite、FTS5、Qdrant、 BGE-M3 与 RRF 的组合：稀疏检索和元数据管理保证轻量稳定， 向量检索负责语义召回，多路结果通过融合策略形成更可靠的论文候选集。

统一论文对象

把标题、摘要、作者、年份、来源、引用关系等信息组织为稳定的文献对象，方便系统和 Agent 共同使用。

🔎

混合检索

结合关键词、语义向量与结果融合，兼顾精确匹配、概念召回与复杂科研问题的表达能力。

🔗

引用扩展

通过引用与被引用关系追踪上下游工作，让文献搜索可以从列表检索升级为研究脉络探索。

算法创新：从固定流程到自主决策

真正困难的学术搜索不是把几个工具调用顺序串起来，而是持续判断： 搜什么、扩什么、何时停止、下一步沿哪条文献链路继续。

PaperScout 将学术搜索建模为多轮决策问题，并通过 PSPO 对 sequence-level 的 Agent 行为进行优化。 它关注完整检索轨迹的质量，而不是只优化局部 token 输出。

  • 1
    Search 根据研究问题生成检索查询，围绕候选论文逐步建立初始证据池。
  • 2
    Expand 沿引用、被引与相关工作关系扩展候选集合，让 Agent 能主动追踪研究谱系。
  • 3
    Stop 在证据足够或继续搜索收益下降时收束检索，把开放探索转化为可交付结果。

智能体应用：进入真实研究工作流

当底层 API 和搜索 Agent 具备可复用能力后，关键问题变成如何进入研究者日常工作。 Lewen-Agent / 学术乐问 将 Lewen-API 与 PaperScout 组织成可直接使用的科技文献检索 Agent， 让算法和 API 变成研究者可以操作的实际系统。

学术鲁班则面向科研 Skills 的管理、上传、下载与共建。当前已有 academic-search、figure-plot 等代表性科研 Skills， 后续可继续扩展论文阅读、论文写作和更多面向科研流程的能力单元。
