TL;DR： ScoutBench 是面向 Agentic Paper Search 的过程奖励评测与训练基准。 它的核心出发点是：科技文献检索天然具有过程奖励——Agent 每一步工具调用都产生可观测的中间结果， 因而 Paper Search 不只是应用任务，更是研究 Agentic RL 的天然训练场。

背景与动机

随着 Agentic AI 从通用问答走向专业领域问题求解，智能体对科学知识获取能力的依赖越来越强。 Science Agent、Medical Agent、Chemistry Agent、Materials Agent、Time Series Research Agent、 NeoResearch 和 CastClaw 等领域智能体，在执行复杂任务时都需要持续获取最新、可信、可引用的专业知识。 科技文献是这些知识最重要的载体之一，因此 Paper Search Agent 将成为未来领域智能体的重要基础技能。

然而，现有论文检索基准大多关注最终检索结果——给定 query 返回论文列表，再计算 Recall、Precision 或 nDCG。 这类设置难以评估一个 Agent 在多轮检索过程中的行为质量，也难以支持过程奖励、工具调用策略和信用分配研究。 另一方面，现有 Agentic RL benchmark 往往依赖稀疏最终奖励，难以提供稳定、可解释的 step-level feedback。

科技文献检索天然具有过程奖励。 一次 Search 可以返回新的候选论文，一次 Expand 可以沿引用网络发现相关工作， 一次 Filter 可以剔除不满足约束的论文，一次 Rerank 可以提升结果排序质量。 这些中间行为都可以根据新增论文的相关性、新颖性、约束满足程度和工具调用成本进行量化评价。 因此，Paper Search 是研究 Agentic RL 的天然训练场。

核心定位

ScoutBench 的定位不是普通的 paper retrieval benchmark，而是同时服务两个研究社区： 对于信息检索和科技文献挖掘社区，提供复杂约束下的多轮学术检索评测环境； 对于 Agentic RL 和 LLM Agent 社区，提供一个具有自然过程奖励的交互式环境， 可用于研究 process reward、tool-use policy、sequence-level optimization、credit assignment 和 long-horizon exploration。

IR Community

复杂约束多轮检索评测

超越一次性关键词检索，提供含方法约束、领域约束、排除条件、时间范围等复杂语义需求的多轮检索评测场景。

Agentic RL Community

自然过程奖励训练环境

利用文献检索中天然存在的 step-level 可观测中间结果，为过程监督、信用分配和长程探索提供稳定训练信号。

任务定义

在 ScoutBench 中，每个任务实例由一个复杂学术检索需求组成。 该需求不是简单关键词，而是包含研究主题、方法约束、领域约束、排除条件、时间范围、数据集偏好或应用场景等复杂语义。 Agent 需要在多轮交互中维护当前 paper pool，根据已有结果决定下一步行动， 并在有限工具调用预算下动态决定搜索什么、扩展哪篇论文、是否继续探索、是否切换方向、何时停止。

典型任务示例

  • Q1
    领域约束 + 架构排除 找出将强化学习用于蛋白质设计、但不依赖 Transformer 架构的代表性论文。
  • Q2
    时效性 + 代码可用性 找出非平稳时间序列预测中显式建模分布漂移的近年工作，并优先关注有公开代码的论文。
  • Q3
    多模态 + 可解释性 找出医学时序中用于术后 AKI 早期预警的深度学习方法，重点关注多模态 EHR 数据和可解释性建模。

任务结束后，Agent 需要提交最终论文集合，或进一步提交结构化 evidence——核心论文、 方法谱系、数据集、实验结论、局限性和引用链条。系统不仅评价最终结果，也评价每一步检索行为是否有效。

标准化工具环境

ScoutBench 提供标准化的交互环境，包括论文语料库、引用图、工具接口、状态表示、奖励函数和评估协议。 Agent 的核心能力体现在它能否根据当前状态动态组合这些工具，而不是机械执行固定流程。

过程奖励设计

ScoutBench 的关键特色是过程奖励。每一步工具调用后，环境都根据新增信息质量给出 step-level reward， 奖励由多个部分构成。这种设计使 ScoutBench 不仅评价最终结果，还能评价 Agent 的检索轨迹质量。

  • 相关性 新增相关论文奖励：若某步工具调用发现了此前未出现、且与用户需求高度相关的论文，给予正奖励。 相关性由人工标注、检索标签、专家规则或 LLM-as-a-Judge 共同确定。
  • 新颖性 新颖性奖励：若 Agent 重复搜索相同 query、重复扩展同一篇论文，或持续返回已发现的论文， 不给奖励或给予惩罚，鼓励 Agent 避免无效探索。
  • 约束满足 约束满足奖励：复杂学术 query 往往包含细粒度限制，Agent 找到的论文不仅要主题相关， 还要满足排除条件、数据集偏好、代码可用性等约束。
  • 引用覆盖 引用图覆盖奖励：高质量检索不仅要找到单篇论文，还需覆盖关键研究分支、经典基础工作、 代表性后续工作和不同方法路线。基于 citation cluster 或 method lineage 设计覆盖奖励。
  • 工具成本 工具成本惩罚：每次工具调用都有一定成本，鼓励 Agent 在有限预算下高效检索， 而不是无限调用工具以提升召回。
  • 停止决策 停止决策奖励：在充分覆盖后及时停止可获得奖励；过早停止导致召回不足， 或过晚停止造成成本浪费，均应受到惩罚。

评估指标

ScoutBench 的评估指标包括最终结果质量和过程行为质量两类， 后者更能体现 ScoutBench 区别于传统 IR benchmark 的独特价值。

Final Result Quality

最终结果质量

  • Recall / Precision / F1
  • Recall@k · nDCG
  • LLM Relevance Score
  • Constraint Satisfaction Rate
  • Citation Coverage
Process Behavior Quality

过程行为质量

  • Tool-Call Efficiency
  • Average Reward per Action
  • Redundant Action Ratio
  • Search-Expand Balance
  • Trajectory Diversity
  • Early-Stage Recall Gain
  • Stopping Quality

数据与轨迹资源

为了支持 Agentic RL 研究，ScoutBench 不仅发布静态 query 和 ground truth， 还提供标准化 trajectory logs——每条轨迹包含完整的检索过程记录，可直接用于多种训练范式。

Trajectory Format

标准化轨迹格式

每条轨迹包含：user query · observation · Agent reasoning · tool call · 工具返回结果 · 新增论文集合 · step reward · 当前 paper pool · 最终提交结果 · 最终 reward。

Research Use Cases

支持多种研究范式

SFT：学习高质量检索行为。Offline RL：利用已有轨迹优化策略。 Reward Modeling：学习过程奖励模型。Process Supervision：研究中间反馈对多轮智能体训练的提升效果。

轨迹字段 内容说明 主要用途
query 用户提出的复杂学术检索需求，包含多维约束。 SFT · Reward Modeling
observation 每步工具调用后环境返回的可观测结果。 Offline RL · Credit Assignment
tool_call Agent 当前步骤的行动：Search / Expand / Read / Filter / Rerank / Stop。 Tool-Use Policy Learning
step_reward 环境对当前步骤的即时过程奖励，多维度构成。 Process Supervision · Reward Modeling
paper_pool 当前已发现的候选论文集合及其状态。 State Representation Learning
final_reward 最终提交结果的综合质量评分。 Offline RL · SFT 过滤

预期贡献

ScoutBench 的预期贡献覆盖 benchmark 设计范式、Agentic RL 基础设施和领域智能体知识获取能力三个层面。

  • 01
    新的 Benchmark 视角 将科技文献检索建模为具有过程奖励的多轮智能体任务，而不是一次性检索排序任务。 突破了传统 IR benchmark 仅评价最终结果的局限。
  • 02
    标准化 Agentic RL 环境 使研究者能够系统研究过程奖励、工具调用策略、序列级优化、信用分配和长程探索， 提供稳定、可解释的 step-level 训练信号。
  • 03
    科学知识获取基础设施 为未来 Science Agent、Medical Agent、NeoResearch、CastClaw 等领域智能体 提供可训练、可评测、可复用的 Paper Search Skill 基座，支撑 Agentic AI 时代的知识获取能力建设。
一句话总结： ScoutBench 旨在将 Paper Search 从静态信息检索任务升级为可交互、可反馈、可训练的 Agentic RL 环境。 它利用科技文献检索天然存在的过程奖励，为多轮工具调用智能体的过程监督、信用分配和策略优化提供标准化 benchmark， 并进一步支撑 Agentic AI 时代的科学知识获取基础设施建设。
